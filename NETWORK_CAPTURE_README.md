# HTML + 接口抓取融合功能说明

## 功能概述

现在 Save Page WE 插件支持同时抓取 HTML 页面和接口请求（HAR 格式）。

## 核心特性

### 1. 统一控制界面
- 在自动抓取控制台新增"同时抓取接口（HAR格式）"复选框
- 可以单独开启 HTML 抓取，或同时开启 HTML + 接口抓取

### 2. 窗口级监控
- HTML 抓取和接口抓取都支持窗口级自动跟踪
- 自动监控窗口内所有 Tab 的切换和页面跳转
- 新打开的 Tab 会自动附加监控

### 3. 数据同步导出
- 停止监听时自动导出 HTML 快照和 HAR 文件
- HTML 快照保存在指定子目录
- HAR 文件自动下载到默认下载目录

## 技术实现

### 架构

```
┌─────────────────────────────────────────────────────────┐
│           统一控制面板 (capture-control.html)            │
│  ☑ 抓取HTML    ☑ 抓取接口（HAR格式）                    │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   Background Service Worker             │
│  ┌──────────────────────┐  ┌──────────────────────┐    │
│  │  HTML抓取模块         │  │  接口抓取模块         │    │
│  │  - IndexedDB存储      │  │  - HAR存储           │    │
│  │  - MutationObserver   │  │  - Chrome Debugger   │    │
│  └──────────────────────┘  └──────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### 核心文件

1. **manifest.json**
   - 添加 `debugger` 权限
   - 启用 ES6 模块支持 (`type: "module"`)

2. **network-monitor.js**
   - 网络监控核心模块
   - 使用 Chrome Debugger API 监听网络事件
   - 支持 HAR 格式导出

3. **har.js**
   - HAR 格式处理工具
   - 将录制的网络请求转换为标准 HAR 格式

4. **background.js**
   - 导入网络监控模块
   - 处理网络录制相关的消息

5. **capture-control.html/js**
   - 添加接口抓取复选框
   - 同步控制 HTML 和接口抓取
   - 显示网络录制状态

## 使用方法

### 1. 开启监控

1. 在网页上右键，选择"打开自动抓取控制台"
2. 勾选"同时抓取接口（HAR格式）"复选框（可选）
3. 点击"开始监听"按钮

### 2. 监控过程

- HTML 抓取：监听 DOM 变化，自动抓取页面快照
- 接口抓取：捕获所有网络请求（XHR、Fetch、资源加载等）
- 可以在窗口内切换 Tab 或打开新页面，监控会自动跟随

### 3. 停止并保存

点击"停止监听并保存"按钮：
- 自动保存所有 HTML 快照
- 自动导出 HAR 文件（如果开启了接口抓取）
- 显示保存的快照数量和接口数量

## HAR 文件格式

导出的 HAR 文件包含：
- 请求 URL、方法、请求头、请求体
- 响应状态、响应头、响应体
- 时间信息（DNS、连接、SSL、发送、等待、接收）
- 关联的页面信息

## 注意事项

1. **权限要求**：首次使用时，浏览器会请求 debugger 权限
2. **性能影响**：开启接口抓取会略微增加内存占用
3. **响应体大小**：大型响应体可能不会完全保存（受 Chrome 限制）
4. **扩展页面**：自动忽略 chrome-extension:// 页面

## 兼容性

- Chrome 88+ (Manifest V3)
- 支持 HTTP 和 HTTPS 协议
- 自动跟踪窗口内的所有 Tab
